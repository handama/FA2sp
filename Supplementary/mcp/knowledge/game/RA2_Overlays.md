# 覆盖物（overlay）基础知识

覆盖物是地图中的一种特殊结构，因其一般紧贴在单元格上而被称为“覆盖物”。每个单元格只能有一个覆盖物。覆盖物属性包括覆盖物索引（Overlay index）与覆盖物数据索引（Overlay data index），二者共同决定了一个覆盖物的图像。

## 围墙覆盖物

围墙覆盖物在Rulesmd中有`Wall=yes`，它们的覆盖物数据遵循特定的规则：

- 围墙可以拥有多种损坏状态，例如`完好`、`损坏`与`严重损坏`，每一种损坏状态占据16个覆盖物数据，在覆盖物数据中存储在高4位。
- 围墙的连接方向影响覆盖物数据的第四位，围墙可以与相邻的四个方向产生任意连接，具体规则如下：

设损坏状态为 D：

`overlayData = (D × 16) + [ (NW?8:0) + (SE?2:0) + (SW?4:0) + (NE?1:0) ]`

即低 4 位由这四个方向的连接状态累加而成：

- 若西北方向有同种围墙 → +8
- 若东南方向有同种围墙 → +2
- 若西南方向有同种围墙 → +4
- 若东北方向有同种围墙 → +1

不过，在实际使用中，可以直接通过`place_wall()`函数放置围墙，地编会自动处理连接方式
