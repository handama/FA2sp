# 作战小队、特遣部队与动作脚本 （teams, taskforces and scripts）

## 三者的关系

作战小队是地图作者指挥单位执行指定任务的最基本单元。作战小队内部指定了它所使用的特遣部队与动作脚本。作战小队让特遣部队所形成的一群（或一个）单位执行动作脚本，并规定了其他额外属性（比如关联标签可以用于触发事件、忽视阻拦属性可以让作战小队的单位忽视受到的任何攻击）。

特遣部队指定了这个作战小队拥有那些单位。对于一个已经形成的作战小队，它的特遣部队也可被称为小队成员。

动作脚本指定了这个作战小队的单位如何行动。一般情况下，作战小队按照动作脚本的顺序依次执行。

## INI 记录

三者均通过注册表小节+自身小节的方式在地图ini内记录。

- 作战小队的注册表是 `[TeamTypes]`
- 特遣部队的注册表是 `[TaskForces]`
- 作战小队的注册表是 `[ScriptTypes]`

例如：
```ini
[ScriptTypes]
0=01000001
1=01000002
```

其中`key`仅为固定排序，无实际意义；`value`为对应对象的`ID`。

自身小节的小节名与`ID`相同，例如，上述`01000001`动作脚本的ini小节即为`[01000001]`。

三者的`ID`均为全局唯一，既不能与其他同类对象冲突，也不能与任意其他类型的对象（包括作战小队、特遣部队、动作脚本、触发、触发事件、触发行为、标签、AI触发）相冲突。

以上信息仅作了解，在实际lua脚本中，使用`get_teams()`、`get_task_forces()`、`get_scripts()`即可分别获取三者的所有对象的`ID`

## 特遣部队成员

特遣部队的成员可以是车辆、步兵或飞行器。全部可用的单位ID分别可以通过`get_values("VehicleTypes", "rules+map")`、`get_values("InfantryTypes", "rules+map")`、`get_values("AircraftTypes", "rules+map")`获取。
