# 地图坐标与视角

## 地图坐标系

红色警戒2地图使用二维网格（Grid）表示地形。

每个地图单元格（Cell）都有唯一的坐标：

* X：水平坐标。
* Y：垂直坐标。

例如：

```text
(0,0)  (1,0)  (2,0)
(0,1)  (1,1)  (2,1)
(0,2)  (1,2)  (2,2)
```

地图包含两个重要参数：

* Width：地图逻辑宽度 （对应lua函数的`width()`）
* Height：地图逻辑高度 （对应lua函数的`height()`）

这两个参数描述的是实际可见、可编辑的地图区域尺寸。

地图底层实际存储为一个正方形网格，其边长为：

```text
IsoSize = Width + Height
```
（对应lua函数的`iso_size()`）

因此所有地图单元格（Cell）坐标均位于：

```text
0 < X < IsoSize
0 < Y < IsoSize
```

但并非所有坐标都属于实际地图。地图的中心点是`(IsoSize/2 , IsoSize/2)`

游戏中的有效地图区域，是该正方形的一个矩形区域，这个矩形的边与正方形的边有45°夹角。

对于：

```text
Width = W
Height = H
```

有效地图区域四个顶点坐标为：

西北角（NW）：`(1, W)`

东北角（NE）：`(W, 1)`

西南角（SW）：`(H, W + H - 1)`

东南角（SE）：`(W + H - 1, H)`

```text
(W + H - 1, H)
```

实际能够编辑和游玩的地图区域，即位于这四个顶点围成的矩形内部。

## 游戏视角

红色警戒2采用正交视角。

例如：

```text
数据坐标：

(0,0) (1,0) (2,0)
(0,1) (1,1) (2,1)
(0,2) (1,2) (2,2)

显示效果：

      (0,0)
   (0,1) (1,0)
(0,2) (1,1) (2,0)
   (1,2) (2,1)
      (2,2)
```


## 地图坐标轴向与玩家视角方向的关系

地图内部的坐标轴（X轴、Y轴）与玩家在游戏画面中看到的左右/上下方向是**旋转 45 度**的关系。

玩家视角的方向与地图坐标轴向的关系如下：

| 玩家视角 | 对应的地图坐标方向 |
|----------|------------------|
| **左右（西-东）方向** |  `Y = C - X` 方向（`ix + iy = const`） |
| **上下（北-南）方向** | `X = Y` 方向（`ix = iy`） |


## 邻接关系

一个单元格通常有 8 个邻居：

```text
      (-1,-1)
   (-1,0) (0,-1)
(-1,1) (自身) (1,-1)
    (0,1) (1,0)
       (1,1)
```

红警2地图坐标方向与游戏实际显示方向（屏幕方向）并不一致。

坐标增加方向如下：

```text
(X+1, Y)   -> 东南（SE）
(X-1, Y)   -> 西北（NW）

(X, Y+1)   -> 西南（SW）
(X, Y-1)   -> 东北（NE）
```

对应方向：

```text
          北

    西北      东北

西        自身        东

    西南      东南

          南
```

因此：

* X轴沿西北－东南方向延伸
* Y轴沿东北－西南方向延伸

用户所描述的方向，如没有明确说明，均为屏幕方向。

## 地图高度

每个单元格包含一个高度值（Height），范围为0~14。
