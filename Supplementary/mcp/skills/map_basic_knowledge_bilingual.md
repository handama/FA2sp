# 红色警戒2术语中英对照表（Red Alert 2 Terminology Chinese–English Glossary）

## 基础类型

| 中文 | English | 别称 | 说明 |
|------|-------|------|------|
| 地图类型 | Theater | 气候 | 地图地表的基本类型，包括温和、雪地、城市、新城市、月球、沙漠六种 |
| 温和 | TEMPERATE | 草地、温带 | 地图类型之一 |
| 雪地 | SNOW |  | 地图类型之一。一般情况下，其他地图类型的建筑图像均一致，而雪地类型则需要特殊的覆雪图像 |
| 城市 | URBAN | 二班、旧城市 | 地图类型之一 |
| 新城市 | NEWURBAN | 牛二班、新城 | 地图类型之一 |
| 月球 | LUNAR | 太空 | 地图类型之一 |
| 沙漠 | DESERT |  | 地图类型之一 |
| 游戏对象 | Object | 对象 | 所有游戏内的实体对象 |
| 地形 | Tile | ground、land、terrain、地表 | 组成地图的地形，如悬崖、公路等。在文件层面上，每个地形文件存储的都是一个地形；在编码上，每个地形都有一个独特的地形索引 |
| 地形组 | Tile set |  | 同类地形组成的分组，如多种不同方向的公路组成公路组。在编码上，同一地形组的地形索引是连续的 |
| 单元格 | cell | 格子、单元、地形块、tile | 游戏地形的最小单位，一个地形往往由多个地形块组成 |
| 覆盖物 | Overlay | 覆盖图、覆盖 | 一种放置在单元格上的结构，常见的有金矿、围墙、桥梁等。一般不被称为游戏对象 |
| 覆盖物数据 | OverlayData | 覆盖图数据 | 规定了某种覆盖物的变体，比如破损的围墙 |
| 建筑 | Structure | 建筑物、building | 固定的、可以占据多个单元格的游戏对象 |
| 步兵 | Infantry |  | 可动的游戏对象，一个单元格内最多可以有三个步兵 |
| 车辆 | Unit | vehicle、坦克、战车 | 可动的游戏对象，包含舰船（一般而言，直升机或空艇都属于车辆） |
| 飞行器 | Aircraft | 飞机 | 可动的游戏对象 |
| 科技类型 | Techno | | 包括建筑、步兵、车辆、飞行器，游戏内玩家只能操作科技类型 |
| 单位 | Footclass | unit | 所有可动的科技类型，包括步兵、车辆、飞行器 |
| 地形对象 | TerrainType | 树木、tree、terrain | 不可动的游戏对象，一般包括树木、矿井、路灯等。虽然其中文中有“地形”（英文中有“Terrain”），但它并不是地形 |
| 污染 | Smudge |  | 不可动的游戏对象，一般包括弹坑与焦土（crater and burnt） |
| 单元标记 | Celltag |  | 放置于单元格上，表明这个单元格关联到了某一特定标签 |
| 基地节点 | base node | 节点 | 仅用于单人地图，用于规划AI所属方如何以及在哪里建造建筑 |
| 隧道 | Tube | tunnel | 放置于地图两个单元格上的特殊结构，可以让单位在两个单元格之间以地下潜行的方式移动 |
| 路径点 | Waypoint | 航点 | 放置于地图单元格上的标记，一般用于小队或触发，如在让小队成员移动到路径点，在路径点释放超级武器 |
| 触发 | Trigger | 触发器 | 任务编辑最核心的功能，可以在满足特定事件时执行特定行为，一个完整的触发一般包含触发本体、标签、触发事件和触发行为 |
| 标签 | Tag |  | 标签控制触发的重复类型，同时可以关联到科技类型、作战小队上，成为触发与特定科技类型间的中介 |
| 作战小队 | Team | TeamType、小队、团队 | 一群单位，可以由地图作者控制其行为 |
| 特遣部队 | TaskForce | 特遣 | 一个作战小队的小队成员 |
| 动作脚本 | Script | ScriptType、脚本 | 控制作战小队的成员行动的具体方法 |
| AI触发 | AI Trigger | AI | 控制计算机玩家在何种情况下生成何种作战小队 |
| 局部变量 | Local variables |  | 记录在当前地图内的一组状态 |
| 全局变量 | Global variables |  | 记录在全局的一组状态，可以跨关卡传递 |
| 文本标签 | CSF | 翻译、文本、CSF Label、CSF 标签  | 文本字符串资源标签，用于关联游戏中的显示文本。规则文件中引用文本标签，游戏再从字符串表（`.csf`文件）中查找对应的翻译文本并显示 |
| 国家 | Country |  | 某个抽象的国家，对应`[Countries]`注册表 |
| 所属方 | House | 所属、玩家、阵营 | 一场游戏内，归属于不同玩家（包括人类和电脑）操作的国家，对应`[Houses]`注册表。多个所属方可以引用同一个国家，可以将所属方理解为国家的实例。触发、小队中填写的国家参数，最终往往会映射到第一个引用这个国家的所属方 |


## 触发系统

| 中文 | English | 别称 | 说明 |
|------|-------|------|------|
| 触发事件 | Event | 事件、条件 | 一个触发想要执行所必须满足的事件，一个触发可以有多个事件，必须同时全部满足才能执行 |
| 触发行为 | Action | 行为、触发结果 | 一个触发所执行的全部操作，按照顺序依次执行 |
| 关联标签 | Attached tag |  | 关联到科技类型、作战小队上的标签，从科技类型、作战小队的角度讲，被称为它们的关联标签 |
| 下级触发 | Attached trigger | 关联触发 | 一个触发下所关联的另一个触发，下级触发与主触发共享主触发的标签 |
| 重复类型 | Repeat type | 重复 | 标签的属性，决定关联该标签的触发能够执行的次数。分别为 `0 - 任一满足，单次`、`1 - 所有满足，单次`、`2 - 任一满足，重复` |
| 国家 | Country | 所属方、所属 | 触发的所属国家。在实际游戏中，会选取第一个引用该国家的所属方 |

## 作战小队
		
| 中文 | English | 别称 | 说明 |
|------|-------|------|------|
| 招募 | Recruite | 重组、征召 | 建立小队时，将同所属方的闲置单位或更低优先级小队成员加入该小队，而不是全新生产的行为。招募与重组基本意义一致，同时出现时，招募指招募闲置单位，而重组指招募其他小队成员 |
| 国家 | Country | 所属方、所属 | 作战小队的国家。在实际游戏中，会选取第一个引用该国家的所属方 |
| 关联标签 | Attached tag | 标签 | 关联到该小队的标签，会关联到该小队的所有成员上 |
| 特遣部队 | Taskforce | 小队编制 | 作战小队使用的特遣部队，定义单位编组。 |
| 星级 | Veteran level | 等级、经验 | 小队的初始星级，决定单位的经验等级。 |
| 优先级 | Priority | 优先 | 较高优先级值的小队，可以从较低优先级值的小队类型中招募小队成员 |
| 最大 | Max | 最大数量 | AI所属方的此小队存在个数的最大值 |
| 科技等级 | Tech level |  | 无效 |
| 运输机起始 | Transport waypoint | | 指定运输机空投小队的起始路径点 |
| 使用运输机起始 | Use transport origin | | 使用运输机起始路径点 |
| 分组 | Group | 编组、编号 | 小队的分组编号 |
| 小队出现于 | Waypoint | 出现路径点 | 小队出现的位置路径点。 |
| 心控结果 | Mind control decision |  | 仅对AI所属方的小队有效。指的是该小队中包含有心控能力的单位时，如何处理其心控对象，不是这个小队被心控后该怎么做 |
| 可装载 | Loadable | 允许装载 | 运输工具能否装载步兵 |
| 预装载小队 | Full | 满载 | 小队成员是否以满装载状态刷出来 |
| 骚扰效果 | Annoyance | 骚扰 |  |
| 保护缓慢单位 | Guard slower | 护送 | 小队的聚集地会倾向于移动较慢的成员 |
| 忽视分组 | Recruiter | 重组 | 招募成员时，是否忽略单位原有分组，强制进行重组 |
| 自动建造 | Auto create | 自动创建、考虑建造 | 只与小队的招募有关 |
| 预建造 | Prebuild | 预建立 | 无效功能 |
| 增援部队 | Reinforce | 增援 | 建立小队后，小队内成员损失，就会马上补充损失的成员 |
| 运输机空投 | Droppod | 空投、xargo plane | 小队所有成员是否会坐运输机部署 |
| 平民哀叫 | Whiner | 哀叫 | 无效功能 |
| 用后即弃 | Loose recruit |  | 无效或未知 |
| 侵略部队 | Aggressive |  | 当小队成员遭遇敌人，且敌人在小队所有成员的GuardRange范围内时，小队成员会跳过当前移动脚本并去攻击敌人 |
| 忽视阻拦 | Suicide | 自杀式攻击、忽略阻拦 | 小队成员遭遇敌人时会无视敌人并继续完成脚本 |
| 只为传送 | On transport only |  | 无效 |
| 躲避威胁 | Avoid threats |  | 小队前进时自动识别危险区域并尝试避开威胁 |
| 无视离子风暴 | Ion immune | 离子免疫 | 无效功能 |
| 运载器下客后返回 | Transports return on unload | 卸后返回 | 运载工具卸下乘客后是否自动返回装载的位置 |
| 小队成员可被重组 | Are team members recruitable | 允许重组 | 为no时，小队单位的`无条件招募`为0；为yes时，小队单位的`无条件招募`为1 |
| 用于基地防御 | Is base defense | 防御小队 | 小队是否用于基地防御任务 |
| 仅攻击敌对方 | Only target house enemy |  | 小队是否把目标限制于有仇恨且仇恨最大的所属方 |

## 特遣部队

| 中文 | English | 别称 | 说明 |
|------|-------|------|------|
| 分组 | Group | 编组、编号 | 使用该特遣部队的小队成员的分组编号，只有在对应小队的分组为-1时有效 |
| 单位 | Footclass | unit | 该特遣部队的成员 |

## 动作脚本

| 中文 | English | 别称 | 说明 |
|------|-------|------|------|
| 行为 | Action | 动作行为、脚本行为、脚本行 | 脚本内的一个行为，可以指挥单位执行一个特定命令 |
| 参数 | Parameter | Param | 行为的参数，可以调节行为的执行 |
| 额外参数 | Extra Parameter | | 行为的额外参数，一个行为始终只能有一个参数，但是某些行为可以将参数拆分为高16位与低16位，记高16位为额外参数，低16位为基本参数 |
